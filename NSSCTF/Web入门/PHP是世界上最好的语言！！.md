这道题目考的是最直接、最基础的RCE（远程代码执行）知识点。
以下是这道题背后的**核心考点**：

### 1. PHP基础语法考点（格式）
- **语句必须以 `;` 结尾**：PHP中分号表示语句结束，漏掉会报语法错误。
- **字符串引号**：双引号 `" "` 可以解析变量，单引号 `' '` 不解析。这里用双引号或单引号都行，因为 `cat /flag` 里没有变量。
- **必须闭合标签**：虽然在纯PHP文件中可以不写 `?>`，但若题目给了输入框直接填代码，通常后台会拼接为 `<?php [你输入的内容] ?>`，所以只需填内部语句即可。

### 2. 核心函数 `system()` 考点
- **功能**：PHP内置函数，用于执行**外部系统命令**，并将命令的**输出结果直接打印**到页面（浏览器/终端）。
- **同类函数对比**（常考）：
  - `exec()` —— 执行命令，但**只返回最后一行**结果（不直接打印）。
  - `shell_exec()` —— 执行命令，**返回全部结果**（不直接打印，需要用 `echo` 输出）。
  - `passthru()` —— 执行命令，**直接输出原始二进制或全部结果**。
  - 反引号 `` ` `` —— 是 `shell_exec()` 的语法糖，如 `` `cat /flag` ``。
- **这道题为什么用 `system`？** 因为要**直接看到flag内容**，用 `system` 最省事，一行代码搞定回显。

### 3. Linux系统命令考点
- **`cat`**：Linux下用于**读取文件内容并输出到终端**的常用命令。
- **`/flag`**：**绝对路径**，表示Linux根目录下的 `flag` 文件。CTF题目的标准配置，flag文件通常放在根目录或 `/var/www/html` 下。

### 4. 安全与配置考点（为什么能成功？）
- **`disable_functions` 未禁用**：`system()` 属于高危函数，真实生产环境通常会在 `php.ini` 中用 `disable_functions` 禁用它。这道题没有禁用，说明环境是特意开放的。
- **`open_basedir` 未限制**：该配置限制PHP能访问的目录。若限制了根目录，就无法读取 `/flag`。题目未设置，所以能直接读取。
- **文件权限可读**：Web进程（如www-data）对该文件有**读取权限**。

### 5. RCE漏洞利用的本质考点
题目通常会在后台使用类似以下代码接收输入：
```php
<?php
eval($_POST['cmd']);   // 或者 system($_GET['cmd']);
?>
```
提交 `system("cat /flag");` 后，相当于**将系统命令注入到了PHP代码执行流中**，从而拿下服务器敏感文件。这道题考察的就是**命令执行漏洞**的最基本利用方式。

---

**总结**：这道题就是用PHP的 `system()` 函数执行Linux的 `cat` 命令读取根目录下的flag文件，考察了**PHP语法、高危函数、Linux基础命令和RCE入门**这四个最基础的Web安全知识点。
