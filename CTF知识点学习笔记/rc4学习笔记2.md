# 🧩 RC4流加密算法学习笔记

## 第一章：核心原理速记

RC4的本质是**根据密钥生成一串伪随机数（密钥流）**，然后利用**异或（XOR）的对称性**进行加解密。

### 1.1 核心公式（死磕三条）
- **加密**：`密文 = 明文 ⊕ 密钥流`
- **解密**：`明文 = 密文 ⊕ 密钥流` （和加密完全一样！）
- **关键**：密钥流由密钥决定，**不知道密钥就无法生成密钥流**。

### 1.2 两大步骤（KSA + PRGA）
可以把RC4想象成一个**发牌员（密钥流生成器）**：

| 阶段 | 名称 | 通俗理解 | 算法动作 |
| :--- | :--- | :--- | :--- |
| **阶段1** | **KSA** <br>（密钥调度算法） | **洗牌**：根据你输入的密码（Key），把一副初始有序（0~255）的扑克牌打乱。 | `j = (j + S[i] + key[i%len]) % 256` <br>交换 `S[i]` 和 `S[j]` |
| **阶段2** | **PRGA** <br>（伪随机数生成算法） | **翻牌**：每次需要加密一个字节，就翻出一张牌，把牌面的数字作为密钥流。 | `i = (i+1)%256`, `j = (j+S[i])%256` <br>交换 `S[i]` 和 `S[j]` <br>**密钥流** = `S[(S[i]+S[j])%256]` |

## 第二章：RC4在内存中的存储结构（考点重灾区）

在C语言中，RC4上下文（Context）通常定义为结构体。**CTF逆向题常把内存快照丢给你，你必须知道怎么解析！**

```c
struct rc4_state {
    unsigned char i;      // 索引1，占1字节（但为了对齐常占4字节）
    unsigned char j;      // 索引2，占1字节（但为了对齐常占4字节）
    unsigned char S[256]; // S盒，256个字节
};
```
**关键注意点（新手必挂）**：
- 如果IDA显示 `_DWORD *v9`（4字节指针），说明结构体里每个成员占**4个字节**。
- 此时内存布局为：`[i][填充][j][填充][S[0]][S[1]]...`。
- **提取S盒时**：如果dump出的是一维字节数组 `a1`，取S盒第n个元素要写成 **`a1[4 * n]`**（跳过多余的填充字节）。

## 第三章：CTF常见题型与出题人“陷阱”

| 类型 | 题目特征 | 难点 | 破解思路 |
| :--- | :--- | :--- | :--- |
| **标准型** | 直接给出明文/密文和硬编码的Key | 无 | 直接用标准RC4跑一遍即可。 |
| **内存快照型** | 不给Key，只给了一串超长的内存数组（本题就是） | 需要从内存中解析出当前的 `i`, `j`, `S盒` | 跳过KSA洗牌阶段，直接从PRGA阶段开始生成密钥流。 |
| **二次异或嵌套型** | RC4加密后，还异或了一个固定值（如 `^0x22`） | 忽视了外层异或，直接解RC4得到乱码 | **脱壳原则**：先对密文异或固定值，还原出真正的RC4密文，再解RC4。 |
| **魔改RC4型** | 修改了KSA或PRGA中的常量（如把 `+` 改成 `-`，或修改了魔数） | 标准脚本跑不出来 | 必须对照汇编代码，魔改Python脚本中的对应算法步骤。 |

## 第四章：通用解题方法论（四步走）

不管题目怎么变，严格按照下面4步排查，绝不失手：

1.  **静态分析找三要素**（在IDA/Ghidra中）：
    - **Key（密钥）**：是硬编码字符串还是动态生成？
    - **Cipher（密文）**：存放密文的全局数组地址。
    - **异或层**：检查RC4调用之后，是否还有 `^=` 操作。

2.  **判断题型**：
    - 有Key → 标准型。
    - 无Key但有一大堆初始化数据 → 内存快照型。

3.  **剥离“外衣”**：
    - 如果有二次异或（如 `^0x22`），先把密文批量异或回来：`raw_cipher = [c ^ 0x22 for c in cipher]`。

4.  **套用代码模板解密**。

## 第五章：CTF通用代码模板（可直接复制使用）

### 模板1：标准型（已知Key和密文）
```python
def rc4_std(key: bytes, data: bytes) -> bytes:
    """标准RC4加解密通用函数（Key已知）"""
    # 1. KSA 初始化S盒（洗牌）
    S = list(range(256))
    j = 0
    for i in range(256):
        j = (j + S[i] + key[i % len(key)]) & 0xFF
        S[i], S[j] = S[j], S[i]
    
    # 2. PRGA 生成密钥流并异或
    i = j = 0
    res = []
    for byte in data:
        i = (i + 1) & 0xFF
        j = (j + S[i]) & 0xFF
        S[i], S[j] = S[j], S[i]
        keystream = S[(S[i] + S[j]) & 0xFF]
        res.append(byte ^ keystream)
    return bytes(res)

# ---- 调用示例 ----
key = b"12345678abcdefghijklmnopqrspxyz"
# 注意：如果题目有外层异或，在这里处理！
cipher_with_xor = bytes([0x9E, 0xE7, ...]) 
cipher_raw = bytes([c ^ 0x22 for c in cipher_with_xor])  # 脱外套

flag = rc4_std(key, cipher_raw)
print(flag.decode())
```

### 模板2：内存快照型（已知i, j, S盒，不知Key）
```python
def rc4_from_memory(i: int, j: int, S: list, cipher: bytes) -> bytes:
    """从内存快照继续运行PRGA解密（跳过KSA）"""
    S = S[:]  # 复制一份，防止修改原数据
    res = []
    for byte in cipher:
        i = (i + 1) & 0xFF
        j = (j + S[i]) & 0xFF
        S[i], S[j] = S[j], S[i]
        keystream = S[(S[i] + S[j]) & 0xFF]
        res.append(byte ^ keystream)
    return bytes(res)

# ---- 调用示例（基于本题） ----
# 假设dump出来的内存数组叫 mem
# 解析：前4字节是i，接着4字节是j（小端序），之后每4字节一个S盒元素
i = mem[0]          # 0x00
j = mem[4]          # 0x00
# 提取S盒（注意：因为C语言中S盒是_DWORD，所以每4字节取一个）
S = [mem[4 + 4*n] for n in range(256)] 

# 脱去0x22外衣
cipher_raw = [c ^ 0x22 for c in byte_array] 

flag = rc4_from_memory(i, j, S, bytes(cipher_raw))
print(flag.decode())
```

## 第六章：实战复盘(example)

**题目源码关键点**：
1. Key = `"12345678abcdefghijklmnopqrspxyz"`
2. RC4加密输入。
3. 加密结果再 `^ 0x22` 与 `byte_14013B000` 比较。

**套用模板1计算**：
```python
key = b"12345678abcdefghijklmnopqrspxyz"
target = [0x9E, 0xE7, ... , 0x2B]  # byte_14013B000

# 脱去0x22外衣
real_cipher = [c ^ 0x22 for c in target]

# RC4解密（对称，直接调用加密函数）
flag = rc4_std(key, bytes(real_cipher))
print(flag) 
# 输出: b'flag{nice_to_meet_you}'
```

## 第七章：避坑指南

1. **长度陷阱**：RC4是流密码，输入多长密文就多长。如果解密出来有乱码，检查是否只需截取前N位（本题只校验前22位）。
2. **大小端问题**：从内存dump出的 `i` 和 `j` 通常是**小端序**（Little-Endian）存储。如果读出来是 `0x01000000`，实际值是 `0x01`。
3. **魔改识别**：如果按标准模板解出来是乱码，立即回去看 **`j = (j + S[i] + key[i % len(key)])`** 这一句。出题人常把 `+` 改成 `-`，或者把 `& 0xFF` 改成 `% 0x100`（虽然等价），甚至改变交换顺序。**永远以汇编指令为准！**
4. **异或层不止一层**：有时候出题人会在输入前也异或一次，记得检查输入传入RC4之前有没有被 `^=` 处理过。
