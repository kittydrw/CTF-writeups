# RSA 学习笔记

## 一、什么是 RSA

RSA 是一种**非对称加密算法**，即加密和解密使用不同的密钥：公钥（加密）和私钥（解密）。

- **安全性**：基于大整数分解的困难性（已知公钥 `n`，很难分解出两个大质数 `p` 和 `q`）。
- **CTF 核心**：题目往往给你公钥和密文，让你想办法恢复私钥或直接解密。
---

## 二、数学基础与公式

### 1. 密钥生成
1. 随机选两个大质数 `p`、`q`；
2. 计算模数 `n = p * q`；
3. 计算欧拉函数 `φ(n) = (p-1) * (q-1)`；
4. 选加密指数 `e`，满足 `1 < e < φ(n)` 且 `gcd(e, φ(n)) = 1`（通常用 65537）；
5. 计算解密指数 `d`，满足 `e * d ≡ 1 (mod φ(n))`，即 `d = e^(-1) mod φ(n)`。

### 2. 加解密
- **加密**：`c = m^e mod n`（`m` 是明文数值，`c` 是密文）
- **解密**：`m = c^d mod n`

### 3. 常用数学工具
- **求模逆**：`pow(e, -1, phi)` (Python3.8+) 或 `gmpy2.invert(e, phi)`
- **大整数开方**：`gmpy2.iroot(x, n)`（返回 `(根, 是否完全开方)`）
- **素数判定/下一个素数**：`gmpy2.next_prime(x)`
---

## 三、CTF 常见考点与攻击方法

### 1. 模数分解（最常见）
- 如果 `n` 较小（512位以下）或能被在线数据库（[factordb.com](https://factordb.com/)）分解，直接得到 `p`、`q`。
- **破解**：拿去 `factordb` 搜，或使用工具 `yafu`、`Msieve` 分解。

### 2. 低加密指数攻击（e 很小）
- 如 `e=3`，若 `m^3 < n`，则 `c = m^3`，直接对 `c` 开三次方。
- 若 `m^3 > n` 但 `m^3 = c + k*n`，可爆破 `k` 再开方。

### 3. 低解密指数攻击（d 很小）
- 使用 **Wiener Attack**（连分数攻击）或 **Boneh-Durfee** 攻击，当 `d < N^0.292` 时可恢复 `d`。
- 工具：`RSAwienerHacker`。

### 4. 共模攻击（相同 n，不同 e）
- 同一明文 `m` 用不同公钥 `(e1, n)`、`(e2, n)` 加密得到 `c1`、`c2`，若 `gcd(e1, e2)=1`，可用扩展欧几里得恢复 `m`。
- 公式：`m = pow(c1, a, n) * pow(c2, b, n) % n`，其中 `a*e1 + b*e2 = 1`。

### 5. 广播攻击（相同明文，不同 n）
- 同一 `m` 用相同 `e`（如 e=3）加密给多个不同 `n`，得到多组 `c`。利用中国剩余定理（CRT）解方程组，得到 `m^e`，再开 `e` 次方。

### 6. 已知高位攻击（Partial Key Exposure）
- 知道 `p` 或 `q` 的一半高位，可通过 Coppersmith 攻击恢复整个因子。
- 工具：`sage` 的 `small_roots` 方法。

### 7. 模数相关攻击
- `p` 和 `q` 相差很小（比如 `|p-q|` 较小），可用 **Fermat 分解**（`n` 接近 `p^2`，直接开方找）。
- 若 `p-1` 或 `q-1` 光滑（因子较小），可用 **Pollard's p-1** 分解。

### 8. 其他变种
- 给出公钥文件（`.pem`），需先提取 `n` 和 `e`（使用 `openssl` 或 `pycryptodome`）。
- 密文不是整数而是文件（如 `.enc`），需读取二进制或 base64 转换。

---

## 四、解题步骤（面对一道 RSA 题）

1. **读取数据**：确定附件格式（文本、`.pem`、`.key`、`.enc`），提取出 `n`、`e`、`c` 等数值。
2. **判断攻击类型**：
   - 尝试 `factordb` 分解 `n`。
   - 检查 `e` 是否很小（如 3、5、17）→ 低指数攻击。
   - 检查 `n` 是否重复出现（共模）或 `e` 有多个。
   - 检查题目是否给了 `p`、`q` 的高位或关系式（如 `p = next_prime(z*y)`）。
3. **实施攻击**：
   - 若分解成功，直接算 `d` 解密。
   - 其他情况，套用对应脚本（Wiener、广播、共模等）。
4. **转换明文**：解密出的数字用 `libnum.n2s(m)` 转字符串，或处理成 flag 格式。

## 六、实战小技巧

- 若 `e` 很大且 `d` 很小，优先考虑 **Wiener Attack**。
- 若 `n` 能分解但位数很大（>1024），用 `yafu` 或 `MSieve` 在本地跑（需时间）。
- 若给了多个密文和指数，查是否满足共模或广播条件。
- 若题目给出了 `p+q` 或 `p-q` 等额外信息，可直接解出 `p`、`q`（解二次方程）。
- 遇到未知格式，先用 `file` 命令（Linux）看文件类型，或直接 `strings` 查看。

## 七、快速代码模板（分解 + 解密）

```python
import gmpy2, libnum

n = ...
e = 65537
c = ...
p = ...   # 从factordb或攻击得到
q = n // p

phi = (p-1)*(q-1)
d = pow(e, -1, phi)          # Python3.8+
m = pow(c, d, n)
flag = libnum.n2s(int(m)).decode()
print(flag)
```

## 八、总结

RSA 在 CTF 中属于“考得最多”的密码题型，核心就是**想方设法分解 `n`**，或者利用参数配置不当（如小 `e`、小 `d`、共模、广播）。掌握常见的攻击套路和工具，配合 Python 脚本，遇到 RSA 题基本能快速解决。

**记住**：先看 `n` 能否分解，这是最快路径；不行再尝试其他专用攻击。
