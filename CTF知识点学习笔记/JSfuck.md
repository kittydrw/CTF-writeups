# JSFuck 学习笔记

## 一、JSFuck 是什么？

JSFuck 是一种**仅使用 6 个字符** `[ ] ( ) ! +` 就能写出任意 JavaScript 代码的编程风格。

> 名字来源于 Brainfuck（一种极简编程语言），但 JSFuck 的代码可以直接在任何 JS 环境中运行，不需要额外编译器。

**一句话理解**：把正常的 JS 代码，变成一堆括号和符号组成的“天书”，但浏览器依然能正常执行。


## 二、核心原理

JSFuck 利用 JavaScript 的**类型转换机制**，用 6 个字符搭积木：

| 表达式 | 等价结果 | 说明 |
|--------|---------|------|
| `![]` | `false` | 空数组取反得 false |
| `!![]` | `true` | 双重取反得 true |
| `+[]` | `0` | 空数组转数字得 0 |
| `[]+[]` | `""`（空字符串） | 数组相加转字符串 |

**以此为基础可以搭建一切**：
- 数字：`!![]+!![]` = `2`（true + true）
- 字符串 `"false"`：`![]+[]`
- 字母 `"a"`：`(![]+[])[+!+[]]` → 从 `"false"` 中取索引 1
- 任意字符：通过 `String.fromCharCode()` 构造

> 💡 实际 CTF 中几乎不需要手动构造，知道原理、会用工具即可。


## 三、CTF 中如何出题

### 3.1 常见出题方式

| 方式 | 说明 | 难度 |
|------|------|------|
| **源码藏 flag** | 将 flag 用 JSFuck 编码后藏在网页源码注释或 `<script>` 中 | ⭐ 签到 |
| **混淆关键逻辑** | 将验证函数、加密逻辑等用 JSFuck 混淆 | ⭐⭐ |
| **结合 eval 执行** | 用 JSFuck 编码 payload，再用 `eval()` 执行 | ⭐⭐ |
| **多层嵌套混淆** | JSFuck 套娃，或结合 AAEncode、JJEncode 等 | ⭐⭐⭐ |

### 3.2 出题人常用“陷阱”

- 在解码结果中混入干扰字符（如 `[]`），需要手动清洗
- 代码以 `[]["filter"]["constructor"](CODE)()` 格式包裹
- 末尾带有 `()` 自执行，去掉后再解码


## 四、CTF 解题标准流程（3 步走）

### 🔍 第一步：识别特征

**JSFuck 三大特征**：
1. **仅包含** `[ ] ( ) ! +` 六种字符
2. 代码超长（一个简单字符可能变成几十个符号）
3. 常见开头如 `(+[]`、`(![]`、`[][]`

> ⚠️ 注意区分 **Jother 编码**（多了 `{}` 两个字符）

### 🛠️ 第二步：解码（二选一）

**方法一：浏览器控制台（最推荐）**
```
1. 按 F12 打开开发者工具
2. 切换到 Console 面板
3. 粘贴 JSFuck 代码，按回车
4. 直接得到解码结果
```

**方法二：在线工具**
- http://www.jsfuck.com/ 
- http://ctf.ssleye.com/jsfuck.html 
- http://www.hiencode.com/jsfuck.html 
- https://lelinhtinh.github.io/de4js （支持多种混淆）

### 🧹 第三步：清洗结果

解码后如果有干扰字符（如 `[]` 混在 flag 里），用正则去除：
```javascript
flag.replace(/\[\]/g, '')
```



## 五、速查表（CTF 实战用）

| 场景 | 操作 |
|------|------|
| 看到一堆 `[]()!+` | → 判断为 JSFuck |
| 想快速解码 | → F12 → Console → 粘贴回车 |
| 控制台卡顿 | → 用在线工具解码 |
| 结果里有 `[]` 干扰 | → 用 `replace(/\[\]/g,'')` 清洗 |
| 代码以 `()()` 结尾 | → 尝试去掉末尾括号再解码 |
| 不确定是不是 JSFuck | → 搜索 `[+[]]` 或 `![]` 特征 |


## 六、一句话总结

> **JSFuck = 只用 6 个符号写 JS。CTF 中见到一堆 `[]()!+`，直接 F12 粘贴到 Console 回车，基本就能拿到 flag。属于“识别特征→工具秒解”的送分题。**
